Part A — Content API (JSONP)

These tests use the same JSONP transport as index.html and target the intermittent getGlobalSettings startup failure.

PENDING A0 · Live endpoint reachability (no cache)

Checks raw JSONP script-load reachability for primary and backup endpoints directly (no cache fallback). Use this first to confirm live browser connectivity.

PENDING A1 · getGlobalSettings (with retry)

Calls getGlobalSettings via JSONP with up to 4 retry attempts and endpoint rotation — exactly as index.html does on startup. Reports latency and which attempt succeeded.

PENDING A2 · getCourses (JSONP)

Verifies the courses endpoint responds correctly and returns structured data.

PENDING A3 · getWebinars (JSONP)

Verifies the webinars endpoint responds correctly.

PENDING A4 · Intermittency detector (5× rapid calls)

Fires getGlobalSettings 5 times in quick succession to expose flaky responses. If any fail, the endpoint is intermittent — the root cause of the startup issue.

PENDING A5 · Failover endpoint check

Tests whether WEBAPP_BACKUP_URL is configured and reachable. Without a backup URL any single endpoint failure causes a full page load failure.

Part B — Polls API

These tests use REST/POST to the Polls Apps Script deployment.

PENDING B1 · Get All Polls

Retrieves all polls from Google Sheets via the Polls API.

PENDING B2 · Save Test Poll

Creates a temporary test poll via POST and confirms it was accepted.

PENDING B3 · Data Persistence (save → retrieve → delete)

Saves a poll with known data, retrieves it, verifies the round-trip matches, then cleans up.

PENDING B4 · Delete Test Poll

Deletes the test poll created by B2 and confirms removal.
